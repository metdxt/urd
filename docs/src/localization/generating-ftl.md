# Generating `.ftl` Files

The `quest` CLI can automatically generate [Project Fluent](https://projectfluent.org/) stub files from your compiled Urd scripts. These stubs contain every dialogue line and menu option as a Fluent message, pre-filled with the source-language text — ready for translators to copy and localize.

## Basic Usage

```bash
quest gen-l10n script.urd
```

This compiles the script, walks the IR graph, and writes a `.ftl` file to the default output directory:

```text
<script_dir>/i18n/en-US/<script_name>.ftl
```

For example, running `quest gen-l10n examples/localization/merchant.urd` produces:

```text
examples/localization/i18n/en-US/merchant.ftl
```

## Custom Output Directory

Use `--output` to write the `.ftl` file to a different location:

```bash
quest gen-l10n script.urd --output path/to/dir/
```

The file name is always derived from the script name (e.g. `merchant.urd` → `merchant.ftl`).

## Generated FTL Structure

The generated file follows a consistent structure:

```ftl
### merchant.ftl — generated by urd
### Run `quest gen-l10n merchant.urd` to regenerate.
### Source: merchant.urd

## label: start
# @fluent variables: $has_potion (urd: has_potion), $price (urd: price), $gold (urd: gold)

merchant-start-line_1 = You step into the Wandering Bazaar.
    Canvas stalls stretch in every direction, lit by swaying lanterns.

merchant-start-line_2 = Welcome! I am Elara, purveyor of fine curiosities.
    Might I interest you in a Health Potion today?

# menu: merchant-start-menu_1
merchant-start-menu_1-browse_the_wares = Browse the wares
merchant-start-menu_1-ask_about_the_bazaar = Ask about the bazaar
merchant-start-menu_1-walk_away = Walk away

## label: browse
# @fluent variables: $has_potion (urd: has_potion), $price (urd: price), $gold (urd: gold), $item (urd: item)

# interpolation: $price
merchant-browse-line_1 = Behold: the Health Potion! I ask { $price } gold — a true bargain.

# menu: merchant-browse-menu_1
merchant-browse-menu_1-buy_it_for_price_gold = Buy it for { $price } gold
merchant-browse-menu_1-try_to_haggle = Try to haggle
merchant-browse-menu_1-never_mind = Never mind
```

### Key observations

- **File header** — triple-hash (`###`) comments identify the file and include a regeneration hint.
- **Label sections** — double-hash (`##`) comments group messages by label.
- **`@fluent` variable listing** — a comment at the top of each label section lists all Fluent variables in scope (both globals and label-local bindings), showing the Fluent alias and the original Urd variable name.
- **Interpolation comments** — single-hash (`#`) comments before a message note which `$variables` appear in that specific message.
- **Hierarchical message IDs** — every message ID follows the pattern `file_slug-label-context`:
  - Dialogue lines: `merchant-start-line_1`
  - Menu options: `merchant-start-menu_1-browse_the_wares`
  - Conditional branches: `merchant-buy-if_1-line_1`
- **Multi-line messages** — Fluent continuation lines are indented with four spaces.

## Fluent Variable Interpolation

String interpolations in Urd dialogue are automatically converted to Fluent variable references:

| Urd source | Generated FTL |
|---|---|
| `"You have {gold} gold."` | `… You have { $gold } gold.` |
| `"Buy it for {price} gold"` | `… Buy it for { $price } gold` |
| `"Path: {inv.gold}"` | `… { $inv-gold }` (dots become hyphens) |

Variables tagged with `@fluent` are also listed in the scope comments even if they don't appear in the message text — translators may need them for plural rules or grammatical patterns that don't exist in the source language.

## Translation Workflow

Once the base `.ftl` file is generated, the translation workflow is straightforward:

1. **Copy** the generated file to a new locale directory:
   ```bash
   cp i18n/en-US/merchant.ftl i18n/pl-PL/merchant.ftl
   ```

2. **Translate** — replace the English values with the target language, using Fluent features like plural rules and selectors as needed.

3. **Run** with the translated locale:
   ```bash
   quest run merchant.urd --locale pl-PL
   ```

### Polish translation example

Here's a snippet showing how a Polish translator might handle pluralization and grammatical case:

```ftl
## label: buy

# interpolation: $gold
merchant-buy-if_1-line_1 = Znakomicie! Mikstura jest twoja. Masz teraz { $gold ->
    [0]     ani grosza — warta każdej monety!
    [one]   tylko jedną złotą monetę.
    [few]   { $gold } złote monety.
   *[other] { $gold } złotych monet.
}
```

The source English message is a simple interpolation (`You now have { $gold } gold.`), but Polish requires four plural forms and different noun endings. The translator adds these directly in the FTL file — no changes to the Urd script are needed.

See [Plural Rules & Grammar](./plural-rules.md) for more on Fluent's pluralization and selector features.

## Regenerating After Script Changes

When you modify dialogue text or add new labels, re-run `gen-l10n` to produce an updated base template:

```bash
quest gen-l10n merchant.urd
```

This regenerates the **source-language** `.ftl` file. It does **not** overwrite translated files in other locale directories — those are maintained by translators independently.

> **Tip:** Diff the regenerated `en-US` file against the previous version to see which message IDs changed or were added. Translators can then update their locale files accordingly.
