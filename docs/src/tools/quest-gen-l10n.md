# Generating Localization

The `gen-l10n` subcommand generates [Project Fluent](https://projectfluent.org/) (`.ftl`) stub files from a compiled Urd script. These stubs contain every dialogue line and menu option as Fluent messages, pre-filled with the source-language text — ready for translators to copy and localize.

## Usage

```bash
quest gen-l10n <script>
```

### Arguments

| Argument | Description |
|----------|-------------|
| `<script>` | Path to the `.urd` script to process |

### Options

| Option | Description |
|--------|-------------|
| `--output <DIR>`, `-o <DIR>` | Output directory for the generated `.ftl` file. Defaults to `<script_dir>/i18n/en-US/` |

## Examples

Generate with default output location:

```bash
quest gen-l10n examples/localization/merchant.urd
```

This writes to `examples/localization/i18n/en-US/merchant.ftl`.

Generate to a custom directory:

```bash
quest gen-l10n examples/localization/merchant.urd --output path/to/l10n/en-US/
```

## What Gets Generated

The generator walks the compiled IR graph and produces one Fluent message for every:

- **Dialogue line** that carries a `loc_id`
- **Menu option** that carries a `loc_id`
- **Inline dialogue** inside menu option bodies

Each message is pre-filled with the source-language text from the script. String interpolation variables (e.g. `{gold}`, `{price}`) are automatically converted to Fluent placeholders (`{ $gold }`, `{ $price }`).

### Generated File Structure

```ftl
### merchant.ftl — generated by urd
### Run `quest gen-l10n merchant.urd` to regenerate.
### Source: merchant.urd

## label: start
# @fluent variables: $has_potion (urd: has_potion), $price (urd: price), $gold (urd: gold)

merchant-start-line_1 = You step into the Wandering Bazaar.
    Canvas stalls stretch in every direction, lit by swaying lanterns.

merchant-start-line_2 = Welcome! I am Elara, purveyor of fine curiosities.
    Might I interest you in a Health Potion today?

# menu: merchant-start-menu_1
merchant-start-menu_1-browse_the_wares = Browse the wares
merchant-start-menu_1-ask_about_the_bazaar = Ask about the bazaar
merchant-start-menu_1-walk_away = Walk away

## label: browse
# @fluent variables: $has_potion (urd: has_potion), $price (urd: price), $gold (urd: gold), $item (urd: item)

# interpolation: $price
merchant-browse-line_1 = Behold: the Health Potion! I ask { $price } gold — a true bargain.

# menu: merchant-browse-menu_1
merchant-browse-menu_1-buy_it_for_price_gold = Buy it for { $price } gold
merchant-browse-menu_1-try_to_haggle = Try to haggle
merchant-browse-menu_1-never_mind = Never mind
```

### Understanding the Output

- **File header**: contains the file name, a regeneration hint, and the source script name.
- **Section comments** (`## label: ...`): group messages by their parent label.
- **`@fluent` variable comments**: list all Fluent variables in scope for that label — both globals (tagged with `@fluent` in the script) and label-local bindings.
- **Interpolation comments**: note which `$variables` appear inside each specific message.
- **Message IDs**: hierarchical, following the pattern `file_slug-label-line_N` for dialogue and `file_slug-label-menu_N-option_slug` for menu options.
- **Multi-line values**: Fluent continuation lines are indented with four spaces.

## Typical Workflow

1. **Write or update** your `.urd` script.

2. **Generate** the English FTL stub:
   ```bash
   quest gen-l10n my_script.urd
   ```

3. **Send to translators**: they copy the generated file and replace English text with their language:
   ```bash
   cp i18n/en-US/my_script.ftl i18n/pl-PL/my_script.ftl
   # Edit pl-PL/my_script.ftl with Polish translations
   ```

4. **Run** with the translated locale:
   ```bash
   quest run my_script.urd --locale pl-PL
   ```

5. **Regenerate** after script changes:
   ```bash
   quest gen-l10n my_script.urd
   ```

> **Important:** The generated file is a **base template** for translators. It does NOT overwrite existing translated files. If you've already translated `pl-PL/my_script.ftl`, running `gen-l10n` again only updates the `en-US` stub. You'll need to manually merge any new or changed messages into your translated files.

## Directory Layout

After generation, your project will typically look like:

```text
my_script.urd
i18n/
  en-US/
    my_script.ftl      ← generated stub
  pl-PL/
    my_script.ftl      ← translator's copy
  ja-JP/
    my_script.ftl      ← translator's copy
```

The `quest run` subcommand auto-discovers this `i18n/` directory when running your script with `--locale`.
